✈️ 旅行夥伴 (Trip Buddy) - 協作式行程規劃

「旅行夥伴」是一個多功能、實時協作的旅行規劃應用程式，基於 React 和 Firebase Firestore 構建，旨在讓您和您的旅行夥伴輕鬆共同管理行程、預算、成員和待辦事項。

✨ 主要功能

實時同步： 所有數據更改會即時同步給所有行程成員。

多模式界面： 支援淺色和暗黑模式切換，提供最佳視覺體驗。

行程詳盡管理： 包含行程規劃、預算管理、成員協作和備註/待辦清單四個主要模組。

拖曳排序： （在 行程規劃 模組中，您可以拖曳活動項目來重新排序。）

自動登入： 應用程式會自動以匿名或提供的認證資訊登入，無需手動註冊。

🚀 快速上手使用教學

1. 儀表板 (Dashboard)

登入與 ID： 應用程式啟動時會自動登入。您的使用者 ID（userId）顯示在頂部導覽列，這是邀請其他成員加入行程的唯一憑證。

新增行程：

點擊網頁中央的 「新增行程」 卡片。

輸入行程名稱、開始日期和結束日期。

點擊 「建立行程」。

選擇行程： 點擊任何一個行程卡片即可進入 「行程細節」 頁面進行規劃。

2. 行程細節頁面 (Trip Detail)

進入行程後，您會看到頂部有四個主要標籤：行程規劃、預算管理、成員協作、備註與待辦。

🔹 行程規劃 (Itinerary)

日期生成： 頁面會根據您設定的開始/結束日期自動生成每天的區塊。

新增活動： 點擊每天區塊右上角的 + 按鈕。

編輯活動： 您可以直接編輯活動的「時間」、「類型」、「活動名稱/描述」和「地點」。

排序： 點擊並拖曳活動左側的 <GripVertical /> 圖標，可以調整同一天內活動的順序，或將活動拖曳到其他日期。

💰 預算管理 (Budget)

總覽： 頂部顯示總計畫預算、總支出和剩餘金額。您可以修改「貨幣單位」。

類別明細：

點擊 「新增預算類別」 來創建新的類別（例如：住宿、交通、餐飲）。

在每個類別中，輸入 計畫金額 和 實際支出，系統會自動更新總覽數據。

點擊垃圾桶圖標 <Trash2 /> 刪除類別。

👥 成員協作 (Member)

邀請成員：

點擊 「邀請新成員」。

輸入您朋友的 使用者 ID（請他們查看他們應用程式頂部的 ID）。

點擊 <Check /> 儲存。

協作： 一旦成員加入，他們將擁有完全的編輯權限，所有人的操作都會實時同步。

移除成員： 點擊成員旁邊的 <Trash2 /> 圖標即可移除（您無法移除自己）。

📝 備註與待辦 (Notes & Tasks)

旅行備註： 使用下方的文字框記錄重要的資訊、緊急聯絡方式或旅行中的隨筆。

待辦清單：

在輸入框中輸入任務，然後點擊 <Plus /> 新增。

點擊任務左側的圓圈即可切換任務的 已完成/未完成 狀態。

3. 基本資訊編輯與刪除

編輯基本資訊： 在 「行程細節」 頁面頂部，點擊 <Edit /> 圖標進入編輯模式，您可以修改行程名稱、開始日期和結束日期。修改後點擊 <Save /> 儲存。

刪除行程： 點擊 <Trash2 /> 圖標會彈出確認視窗，確認後將永久刪除整個行程。

4. 模式切換與教學

暗黑模式： 點擊頂部導覽列的 <Sun /> 或 <Moon /> 圖標，即可切換淺色/暗黑模式。

應用程式教學： 點擊頂部導覽列右側的 <BookOpenText /> 圖標，可隨時重新進入教學頁面。
